Literals or Constants

Lecture Summary

This lecture thoroughly explores the concept of literals in Python, explaining what literals are and their importance as direct constant values used in programs. It details the critical architectural differences between variables assigned direct values (literals) and those assigned computed balances from expressions or user runtime operations. The discussion deepens into the specific rules governing five primary classifications: integer, float, boolean, complex, and string literals.

Additionally, the material introduces fundamental clean-coding features like the deployment of underscore digit-separators for clean data visualization, flags common syntax or runtime exceptions, and evaluates the unique lexical rules applying to strings enclosed inside single, double, or triple quotes.

What are Literals?

In basic terms, literals are direct constant values explicitly written or hardcoded in a script. While variables act as abstract naming containers whose assignments can fluctuate dynamically during execution, literals represent the exact, raw static values themselves.

Core Architecture Contrast:

  • Literal Value Assignment: a = 10 — The number 10 is a concrete integer literal.
  • Computed Assignment: c = a + b — The reference variable c receives a dynamically derived runtime payload, not a pure code-declared literal.

The Five Types of Literals

1. Integer Literals

Integer literals consist of direct, whole-number declarations containing no fractional parts. Python can handle incredibly large integers seamlessly within standard memory boundaries.

  • Readability Separation Rules: Commas (,) cannot be typed out within numbers for structural parsing reasons. Instead, Python uses underscores (_) as visual digit separators.
  • Structural Limits: Underscores must be declared strictly between numeric characters. Placing them at the absolute beginning or the terminal edge of a digit group is invalid.
a = 201
b = 13_52_46_198 # Clean digit separation layout
print(a, b)
Console Output: 201 135246198

2. Float Literals

Float literals describe numbers containing accurate decimal fractions or scientific notation constructs.

  • Scientific Format: Supports exponent markers using uppercase E or lowercase e tokens (e.g., 12E2 represents 12 × 102).
  • Separator Exclusions: Visual underscore lines can separate fractional parts, but they cannot sit immediately adjacent to the structural decimal point marker (._ or _.).
a = 9.75
b = 12E2     # Scientific notation (12 * 10^2)
c = 12_5.6_7 # Float with underscores

print(a, b, c)
print(type(a))
Console Output:
9.75 1200.0 125.67
<class 'float'>

3. Boolean Literals

Boolean parameters contain exact logical constants. The engine supports exactly two precise literals: True and False.

Strict Capitalization Rules: Python is strictly case-sensitive. Writing full lower-case expressions such as true or false will break parser tokenization and trigger exceptions. Furthermore, underlying native integer representations like 1 or 0 are numeric values, not genuine boolean literals.

4. Complex Literals

Complex values combine explicit real components alongside mathematical imaginary structures.

  • Notation Suffix: The specific lowercase or uppercase character j must trail the imaginary component directly.
  • Component Formatting: Both sections can be standard whole integers or independent decimal float entries. Underscores can be embedded cleanly within either part.
a = 4 + 5j
b = 1_2 + 1_4j # Separator line within components
print(a, b)
Console Output: (4+5j) (12+14j)

5. String Literals

Strings capture structured textual blocks, alphabetical words, or sentence chains safely. Python offers three distinct enclosing frameworks:

Single & Double Quotes

Declared via '...' or "..." wrappers. Ideal for single-line text. You can nest them to embed quote characters inside without escaping:

msg = '"Alexa"'

Triple Quotes

Declared via '''...''' or """...""" wrappers. Intended for spanning multi-line text blocks naturally or writing blocks containing mixed internal quote types.

Common Architectural Pitfalls & Errors

Misdeclaring literal values typically encounters two specific runtime or compilation exceptions in Python:

1. SyntaxError (Malformed Literal Structure)

Occurs if literal layouts break fundamental structural lexing rules. For example, putting a trailing separator underscore line right at the terminal boundary of an entry breaks compilation. 

c = 358_ # SyntaxError: invalid decimal literal

2. NameError (Unrecognized Context References)

Occurs when the engine treats an improperly typed or uncapitalized value as an unmapped variable token name. For instance, starting a numeric field with a leading underscore, or forgetting uppercase letters on booleans, fools Python into looking for a pre-declared variable reference. 

d = _125  # NameError: name '_125' is not defined
flag = false # NameError: name 'false' is not defined

Key Takeaways Reference

  • Literal Identity: Hardcoded immutable constant data elements within code statements.
  • Visual Formatting: Use internal underscores to keep large numbers highly legible; never allow them to touch edge boundaries or decimals.
  • Case Guardrails: Always write boolean variables using exact True and False styling to guarantee operational validation.
  • Complex Fields: Suffix imaginary units with the literal j symbol.
  • String Strategies: Leverage single or double wrappers for standard lines, and switch over to triple strings for clean multi-line structuring.
