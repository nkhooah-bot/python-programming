Expressions

Lecture Summary

In our previous lecture, we isolated arithmetic symbols to analyze their isolated behaviors. However, real-world source files rarely contain simple two-operand math lines. Instead, complex application states rely on compounded operations called Expressions.

This lecture studies how Python sequences multiple nested operators within a single statement. We explore the laws of Operator Precedence, detail directional Associativity, and demonstrate how explicit punctuation boundaries allow engineers to override default execution hierarchies to maintain total structural accuracy.

What is an Expression?

An expression is any structurally sound combination of operands (variables, raw literals, or function calls) and operators (+, -, *, /, **) that the execution engine evaluates to return a single data value.

When an expression contains multiple operators, Python relies on a strict internal hierarchy to determine the evaluation order. Changing this order completely changes the resulting data payload.

Operator Precedence

Operator precedence establishes the priority tier of each symbolic token. Operators located on higher tiers are resolved completely before lower-tiered operators are evaluated.

The Arithmetic Precedence Hierarchy (Highest to Lowest):

  1. () – Parentheses (Explicit grouping isolation)
  2. ** – Exponentiation (Power operations)
  3. *, /, //, % – Multiplication, Division, Floor Division, Modulus
  4. +, - – Addition, Subtraction
result = 3 + 4 * 5
print(result) # Multiplication (*) executes before Addition (+)
Console Output:
23

Operator Associativity

When an expression contains multiple operators with the same precedence tier (for example, a line containing both multiplication * and modulus %), Python resolves the tie using directional associativity rules.

Left-to-Right Associativity

Nearly all standard arithmetic operators follow a left-to-right evaluation route. The leftmost operator executes first, and its result is passed to the next operator on the right.

Applies to: +, -, *, /, //, %

Right-to-Left Associativity

Python's exponentiation operator (**) features a unique right-to-left associativity rule. When multiple power symbols are chained together, Python evaluates the rightmost operation first.

Applies to: **

# Left-to-Right Verification: Subtraction
print(10 - 4 - 2) # Evaluates as ((10 - 4) - 2)

# Right-to-Left Verification: Exponentiation
print(2 ** 3 ** 2) # Evaluates as (2 ** (3 ** 2)) → 2 ** 9
Console Output:
4
512

Overriding Hierarchies with Parentheses

Parentheses ( ) have the highest precedence tier in Python. Enclosing an expression in parentheses forces the engine to isolate and evaluate those operations first, regardless of the default precedence rules. 

# Overriding Multiplication Precedence
normal_val = 3 + 4 * 5
forced_val = (3 + 4) * 5

print("Default Precedence:", normal_val)
print("Parentheses Forced:", forced_val)
Console Output:
Default Precedence: 23
Parentheses Forced: 35 
# Overriding Exponentiation Associativity
chained_power = 2 ** 3 ** 2
grouped_power = (2 ** 3) ** 2

print("Default Right-to-Left:", chained_power)
print("Grouped Left-to-Right:", grouped_power)
Console Output:
Default Right-to-Left: 512
Grouped Left-to-Right: 64

Key Takeaways Reference

  • Expression Composition: Expressions blend operators and operands to calculate a single data value without altering source variables in place.
  • Precedence Enforcement: Exponentiation (**) takes priority over products or remainders (*, /, //, %), which outrank basic addition and subtraction (+, -).
  • Directional Exceptions: Standard math operators resolve from left to right, but chained exponentiation (**) triggers from right to left.
  • Parentheses Rule: Explicit grouping via parentheses overrides default precedence and associativity rules, providing a clear tool to maintain predictable outcomes in complex lines of code.
  • Readability Tip: When writing complex expressions, use explicit parentheses to clarify the evaluation order for other programmers, even when Python's default rules would produce the correct result.
