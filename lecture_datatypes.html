Section 1Lecture Summary

This lecture introduces the various fundamental data types available in Python, emphasizing their categories and critical properties. Choosing the right data type is essential for managing how data is stored, manipulated, and accessed efficiently in memory.

We begin by exploring numeric types that hold single scalar values, such as integers, floating-point numbers, booleans, and complex numbers. The discussion then shifts to collection structures, starting with sequence types like lists, tuples, and strings, which support indexed access but differ significantly in their mutability. Finally, we explore unordered collections: sets, which enforce uniqueness without index support, and dictionaries, which store key-value pairs while maintaining insertion order.

Section 2Numeric Data Types (Scalar)

Scalar data types store a single individual value at a given time. Python provides four built-in core types for numerical computations:

int Integer

Represents whole numbers without decimal places. Can be positive, negative, or zero.

x = 10 # Whole number representation

float Floating-point

Represents numbers containing decimal points or fractional parts.

y = 19.25 # Fractional/decimal number

bool Boolean

Represents logical values: either True or False. Internally within Python's execution environment, these can also be expressed numerically as 1 (True) or 0 (False), rendering them highly useful for flags and conditional expressions.

b = True # Logical boolean state

complex Complex Numbers

Used to represent complex mathematical numbers possessing both a distinct real part and an imaginary part, marked by a j suffix.

c = 7 + 9j # Real part: 7, Imaginary part: 9

Section 3Sequence Data Types

Sequences are structured collections of values sharing common behaviors: they are strictly ordered and indexed starting from zero (0, 1, 2, ...). Sequences gracefully support mixed data types within the same collection container.

list Mutable Sequence

Declaring a list requires enclosing elements in square brackets [ ]. Lists are completely mutable; elements can be updated, appended, or permanently wiped out after creation.

L = [2, 4, 6, 8, 10] print(L[0]) # Access by index: outputs 2 L[3] = 7 # Modification allowed: changes 8 to 7 L.append(12) # Adds 12 to the end of list print(L) # Outputs: [2, 4, 6, 7, 10, 12]

tuple Immutable Sequence

Declaring a tuple relies on round parentheses ( ). Tuples appear fundamentally similar to lists but are strictly immutable; they cannot be altered, grown, or shrunken once allocated.

t = (9, 7, 4, 2, 5) print(t[2]) # Access by index: outputs 4 t[2] = 8 # TypeError: 'tuple' object does not support item assignment

string Immutable Character Sequence

Represents a literal ordered sequence of individual characters wrapped neatly in single quotes '...' or double quotes "...". Strings are explicitly immutable.

s = "python" print(s[1]) # Access by index: outputs 'y' s[1] = 'a' # TypeError: 'str' object does not support item assignment

Section 4Set Data Type

A set is an unordered collection of uniquely constrained values. Sets automatically sweep away and ignore any duplicate elements fed into them.

While sets are mutable (you can freely add or remove items), they possess no index support. Therefore, you cannot access or modify individual elements using bracket indices. Sets are declared via curly braces { }.

S = {3, 7, 2, 9, 5, 9} # Duplicate '9' is automatically stripped out print(S) # Outputs unique collection (Order not guaranteed) S.add(8) # Inserts 8 into the set S.remove(7) # Removes 7 from the set print(S)

Section 5Dictionary Data Type

A dictionary stores associations in the structural form of paired keys and values. Dictionaries enforce that all keys must be distinct and unique, though multiple independent keys are free to point to identical duplicate values.

Modern Python dictionaries are ordered, meaning they preserve insertion patterns reliably. They are highly mutable; values linked to valid keys can be swiftly updated. Dictionaries are declared using curly braces containing colon-separated key: value declarations.

d = {'name': 'john', 'roll': 126, 'dept': 'cse'} print(d['name']) # Access value by key: outputs 'john' d['name'] = 'kate' # Modifies value of existing key print(d) # Outputs updated dictionary

Section 6Key Takeaways & Syntax Quick Reference

Mastering Python's built-in data types relies heavily on understanding three structural axis definitions: Mutability, Indexing, and Ordering.

  • Numeric Types: Hold isolated scalar data points (int, float, bool, complex).
  • Sequence Types: Maintained in structural array order and accessed linearly via index; lists stand out as fully mutable, whereas tuples and strings remain rigidly immutable.
  • Sets: Structural bags containing completely unique nodes; mutable but completely index-less.
  • Dictionaries: Storage maps containing key-to-value pathways; keys stay uniquely bound and immutable, whereas targeted values can be updated.

Syntax Convention Reference Sheet

  • Lists: Bounded cleanly by square brackets: [ ]
  • Tuples: Enclosed strictly inside traditional round parentheses: ( )
  • Strings: Captured explicitly inside quotes: "..." or '...'
  • Sets: Outlined clearly by plain curly braces: { }
  • Dictionaries: Set in curly brackets with mapping syntax: {key: value}
