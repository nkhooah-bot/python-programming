Section 1Relational Operators

Relational operators (also known as comparison operators) evaluate the mathematical or empirical relationship between two variables or expressions. When executed, these operators return a strict Boolean metric: either True or False.

Operator Meaning Example Result
< Less than 4 < 5 True
<= Less than or equal to 4 <= 5 True
> Greater than 4 > 5 False
>= Greater than or equal to 4 >= 5 False
== Equal to 4 == 5 False
!= Not equal to 4 != 5 True

Critical Syntax Warning

Python requires a double equals sign == to evaluate equivalence testing. Utilizing a single equals token (=) is reserved strictly for assignment declarations and will trigger systemic parsing errors if misplaced inside a logic gate.

These evaluators are dynamic and can determine scalar status across disparate data primitives:

  • Integers: 4 < 5 evaluates to True.
  • Floats: 4.0 >= 3.5 evaluates to True.
  • Strings: "apple" != "orange" evaluates to True based on lexical divergence tracking.

Section 2Conditional Statements

Conditional statements empower a program to selectively execute alternate code components based on dynamic conditional states evaluated at runtime. This behavior is primarily controlled through coordinated if and else blocks.

if condition:
    # Block of code to run if condition is True
else:
    # Block of code to run if condition is False

Unlike traditional environments using visible brackets, Python requires strict indentation layout configurations (typically utilizing 4 spaces or 1 tab) to demarcate block scope definitions cleanly. Missing alignment tabs will compromise block encapsulation boundaries and trigger runtime exceptions.

Section 3Control Flow

Control flow governs the operational trajectory and execution sequence that instructions follow during runtime operations.

Normal (Linear) Flow: Without logical branch processing paths, code processes line-by-line in a predictable, uniform sequence:

Start —> Statement 1 —> Statement 2 —> Statement 3 —> End

Conditional Flow: When passing through an if...else framework, execution tracks down a split decision matrix:

                  [ Start ]
                      │
                 [Statement 1]
                      │
             ◄ Check Condition ►
                    ╱   ╲
             True  ╱     ╲  False
                  ▼       ▼
         [Statement A]   [Statement B]
                  ╲       ╱
                   ▼     ▼
                  [ End ]

Depending on the evaluated result of the logical statement, only one branch (either the true if path or the false else path) runs before the script resumes standard execution.
