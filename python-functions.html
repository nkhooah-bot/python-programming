Python Functions

Modular Development, Reusability, and Implementation

1. Introduction to Functions

A function is a reusable piece of code designed to perform a specific, smaller task. Whenever you see code written with round brackets—like print() or input()—you are interacting with a built-in function.

  • Task-Oriented: Performs a specific, defined operation.
  • Reusable: Write once, use many times across different programs.
  • Modular: Helps break large applications into manageable modules and subtasks.

2. Writing Your Own Functions

To define a user-defined function, use the def keyword. The structure consists of a header, parameter list, and an indented body.

General Skeleton:

def function_name(parameter_list):
  # Function Body
  return result

3. Practical Example: Cuboid Volume

def volume(length, breadth, height):
  vol = length * breadth * height
  return vol

v = volume(10, 5, 3) # Function Call
print(v) # Output: 150

4. Key Terminology

TermDescription
Header/SignatureThe definition line (def name(...):)
Actual ParametersValues passed during the call (e.g., 10, 5, 3)
Formal ParametersVariables defined in the header (e.g., length, breadth)
ReturnThe statement that sends the result back to the caller

5. Parameter Passing Mechanism

Python uses a unified parameter passing mechanism. When you pass arguments, variables refer to the same object in memory. Note that basic data types (int, float) are immutable; changing them inside a function does not affect the original variable, as a new object is created in memory instead.

6. Best Practices

  • Naming: Use unique names; avoid Python's built-in function names.
  • Flow Control: Understand that function calls transfer control to the function body and return control back upon completion.
  • Modular Design: Use the if __name__ == "__main__": block to allow your code to be imported as a module without executing immediately.
