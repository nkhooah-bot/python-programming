Lecture 16

Rules for Variable Names

1

Lecture Summary

This lecture covers the rules and guidelines Python enforces when naming variables. Understanding these rules is essential because an invalid variable name causes an immediate SyntaxError that stops the program from running. Beyond the hard rules, there are also professional naming conventions that make code readable and maintainable.

2

Key Concepts & Explanations

Rule 1 – Use Meaningful Names (Guideline)

A variable name should clearly describe the data it holds. Meaningful names make code self-documenting.

✅ Good Names
  • product_id — stores a product ID
  • price — stores a price value
  • customer_name — stores a name
❌ Poor Names
  • x — tells you nothing
  • a1 — not descriptive
  • temp — vague purpose

Rule 2 – Alphanumeric Characters and Underscore Only

Allowed Characters

  • Letters: A–Z and a–z
  • Digits: 0–9
  • Underscore: _ (the only special character allowed)
  • Spaces, hyphens, and all other symbols are not allowed

Rule 3 – Must Start with a Letter or Underscore

Important: A variable name cannot begin with a digit. Starting with a number causes a SyntaxError: invalid decimal literal because Python tries to interpret it as a number.

Rule 4 – Cannot Use Python Keywords

Python reserves certain words for its own syntax. You cannot use them as variable names.

Common Python Keywords

  • if else while for return class import None True False
  • Note: False (capital F) is a keyword — false (lowercase) is not, but using it is discouraged.

Rule 5 – Python is Case Sensitive

price, Price, and PRICE are three completely different variables. Always be consistent with capitalisation.

Valid vs Invalid — Quick Reference Table

Example Code

Valid Variable Declarations

valid_names.py 
# All of these are valid variable names
product_id = 153
price = 23.15
customer_name = "James"
_address = "Maseru"
abc123np = 15

print(product_id, price, customer_name)
# Output: 153 23.15 James

Case Sensitivity Demonstration

case_sensitive.py 
price = 10
Price = 20
PRICE = 30

print(price, Price, PRICE)
# Output: 10 20 30  — all three are separate variables

Invalid Name — Error Example

invalid_name.py 
1a = 25
# Output: SyntaxError: invalid decimal literal
# Python sees "1" and expects a number, not a variable name
4

Key Takeaways

1

Variable names must only contain letters, digits, and underscores — no spaces or hyphens.

2

Names must start with a letter or underscore, never a digit.

3

Python keywords like if, for, None cannot be used as variable names.

4

Python is case sensitive — Name and name are different variables.

5

Always choose meaningful names like student_age rather than vague names like x.

6

An invalid name causes a SyntaxError that stops the program immediately.

5

Lecture 17

Taking Input from the User

1

Lecture Summary

This lecture introduces the input() function, which pauses a running program and waits for the user to type something. It explains a critical rule: input() always returns a string, regardless of what the user types. To use numbers, you must explicitly convert the result using int() or float().

2

Key Concepts & Explanations

The input() Function

How it works

  • input() pauses the program and waits for keyboard input.
  • The string passed to input() is the prompt shown to the user.
  • The result is always a string — even if the user types a number.
  • To use the result as a number, you must convert it with int() or float().

Type Conversion

int() — Whole Numbers
  • Converts a string to an integer.
  • Use for age, quantity, year, etc.
  • Fails if the input contains letters or decimals.
float() — Decimal Numbers
  • Converts a string to a decimal number.
  • Use for price, weight, temperature, etc.
  • Accepts values like 3.14 or 99.0.
Common Mistake: Trying to do arithmetic on input without converting it first causes a TypeError. For example, age + 1 will fail if age was obtained directly from input() without conversion.

Taking Multiple Inputs

Using .split()

  • .split() breaks a single input string into multiple parts at each space.
  • You can assign each part directly: name, age = input("Enter: ").split()
  • Always convert to the correct type after splitting when numbers are expected.
3

Example Code

Basic Input and Greeting

basic_input.py 
name = input("Enter your name: ")
print("Hello", name)
# If user types: Alice
# Output: Hello Alice

TypeError Without Conversion vs Correct Conversion

conversion.py 
# ❌ Wrong — age is a string, cannot add 1
age = input("Enter your age: ")
print("Next year you will be", age + 1)  # TypeError!

# ✅ Correct — convert to int first
age = int(input("Enter your age: "))
print("Next year you will be", age + 1)
# If user types: 20 → Output: Next year you will be 21

Rectangle Area Program

rectangle_area.py 
# Program to calculate the area of a rectangle
length = float(input("Enter the length: "))
width  = float(input("Enter the width: "))
area   = length * width
print("Area of the rectangle is:", area)

Multiple Inputs with .split()

multiple_input.py 
# Taking name and age on one line
name, age = input("Enter your name and age separated by space: ").split()
age = int(age)
print("In 5 years,", name, "will be", age + 5)
# If user types: Alice 20
# Output: In 5 years, Alice will be 25
4

Key Takeaways

1

input() always returns a string, regardless of what the user types.

2

Use int() to convert input to a whole number and float() for decimal numbers.

3

Adding a number to an unconverted input causes a TypeError.

4

Use .split() to collect multiple values from a single input line.

5

Always write a clear prompt so the user knows what to enter.

6

Conversion and input can be combined: int(input("Age: ")).

5

Lecture 18

The print() Function

1

Lecture Summary

The print() function is the most commonly used function in Python. It displays information on the screen. This lecture goes beyond basic usage to cover the sep and end parameters that control how values are separated and how lines are terminated, as well as escape characters for formatting text.

2

Key Concepts & Explanations

What Can You Print?

print() accepts

  • Text (strings): print("Hello")
  • Numbers: print(42) or calculations print(10 + 5)
  • Variables: print(name)
  • Multiple values: print("Age:", age) — separated by spaces by default

The sep Parameter

When printing multiple values, Python uses a space as the default separator. You can change this with sep.

Default Separator (space)
  • print("John", "Doe")
  • Output: John Doe
Custom Separator
  • print("John", "Doe", sep="-")
  • Output: John-Doe

The end Parameter

By default, print() adds a newline \n at the end, moving the cursor to the next line. Use end to change this.

Practical use: Setting end=" " allows two print statements to output on the same line. This is useful for building formatted output progressively.

Escape Characters

EscapeMeaningExample Output
\nNew lineMoves to next line
\tTab spaceAdds horizontal indent
\"Double quote inside stringShe said, "Hello!"
\'Single quote inside stringIt's working
\\Literal backslash\
3

Example Code

Basic Printing

basic_print.py 
print("Welcome to Python")    # Text
print(10 + 5)                   # Calculation → 15
a = 10
print("The value of a is", a)  # Text + variable → The value of a is 10

sep and end Parameters

sep_end.py 
# Default separator is a space
print("John", "Doe")               # Output: John Doe

# Custom separator
print("John", "Doe", sep="-")     # Output: John-Doe
print("2025", "06", "01", sep="/") # Output: 2025/06/01

# end parameter — print on same line
print("Hello", end=" ")
print("World")                      # Output: Hello World

Escape Characters

escape_chars.py 
print("Line 1\nLine 2")
# Output:
# Line 1
# Line 2

print("Name:\tJohn")
# Output: Name:    John

print("She said, \"Hello!\"")
# Output: She said, "Hello!"

Formatted Student Profile

student_profile.py 
name = "Alice"
age  = 20
gpa  = 3.75

print("--- Student Profile ---")
print("Name:\t", name, sep="")
print("Age:\t", age,  sep="")
print("GPA:\t", gpa,  sep="")
# Output:
# --- Student Profile ---
# Name:   Alice
# Age:    20
# GPA:    3.75
4

Key Takeaways

1

print() can output text, numbers, variables, calculations, and combinations of all of these.

2

Multiple values are separated by a space by default; use sep to change the separator.

3

By default, each print() ends with a newline; use end to keep output on the same line.

4

\n adds a new line, \t adds a tab inside any string.

5

Use \" or \' to include quote characters inside a printed string.

6

Combining sep, end, and escape characters gives full control over output formatting.

5
